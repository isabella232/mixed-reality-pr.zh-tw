## <a name="eye-gaze-input-design-guidelines"></a><span data-ttu-id="e8a95-121">眼睛輸入設計指導方針</span><span class="sxs-lookup"><span data-stu-id="e8a95-121">Eye-gaze input design guidelines</span></span>

### <a name="benefits-of-eye-gaze-input"></a><span data-ttu-id="e8a95-124">眼睛輸入的優點</span><span class="sxs-lookup"><span data-stu-id="e8a95-124">Benefits of eye-gaze input</span></span>

- <span data-ttu-id="e8a95-127">**輕鬆省力。**</span><span class="sxs-lookup"><span data-stu-id="e8a95-127">**Low effort.**</span></span> <span data-ttu-id="e8a95-128">幾乎不需要任何人為動作。</span><span class="sxs-lookup"><span data-stu-id="e8a95-128">Barely any physical movements are necessary.</span></span> 

- <span data-ttu-id="e8a95-129">**隱含性。**</span><span class="sxs-lookup"><span data-stu-id="e8a95-129">**Implicitness.**</span></span> <span data-ttu-id="e8a95-130">使用者通常會將其描述為「想法閱讀」，使用者的眼睛移動資訊可讓系統知道使用者打算參與的目標。</span><span class="sxs-lookup"><span data-stu-id="e8a95-130">Often described by users as "mind reading", information about a user's eye movements lets the system know which target the user plans to engage.</span></span> 

- <span data-ttu-id="e8a95-131">**替代輸入管道。**</span><span class="sxs-lookup"><span data-stu-id="e8a95-131">**Alternative input channel.**</span></span> <span data-ttu-id="e8a95-132">眼睛可以提供強大的支援輸入，讓您能夠根據使用者的經驗，以多年來體驗使用者體驗。</span><span class="sxs-lookup"><span data-stu-id="e8a95-132">Eye-gaze can provide a powerful supporting input for hand and voice input building on years of experience from users based on their hand-eye coordination.</span></span>

### <a name="challenges-of-eye-gaze-as-an-input"></a><span data-ttu-id="e8a95-138">眼睛作為輸入的挑戰</span><span class="sxs-lookup"><span data-stu-id="e8a95-138">Challenges of eye-gaze as an input</span></span>
<span data-ttu-id="e8a95-139">有許多強大功能，都有很多責任。</span><span class="sxs-lookup"><span data-stu-id="e8a95-139">With lots of power, comes lots of responsibility.</span></span>
<span data-ttu-id="e8a95-140">雖然您可以使用眼睛來建立滿足的使用者體驗，讓您感覺像是 superhero，但您也一定要知道這不是什麼好用。</span><span class="sxs-lookup"><span data-stu-id="e8a95-140">While eye-gaze can be used to create satisfying user experiences which make you feel like a superhero, it is also important to know what it is not good at to appropriately account for this.</span></span> <span data-ttu-id="e8a95-141">以下將討論一些需要考慮的挑戰，以及如何在使用眼睛輸入時解決這些 *挑戰* ：</span><span class="sxs-lookup"><span data-stu-id="e8a95-141">The following discusses some *challenges* to consider and how to address them when working with eye-gaze input:</span></span> 

- <span data-ttu-id="e8a95-142">**您的眼睛是「永遠開啟** 」當您開啟眼睛蓋時，您的眼睛會開始 fixating 環境中的事物。</span><span class="sxs-lookup"><span data-stu-id="e8a95-142">**Your eye-gaze is "always on"** The moment you open your eye lids, your eyes start fixating on things in the environment.</span></span> <span data-ttu-id="e8a95-143">因為您已查看太長的時間，所以會對每個外觀做出反應，並不慎發出動作，因此會產生 unsatisfying 體驗。</span><span class="sxs-lookup"><span data-stu-id="e8a95-143">Reacting to every look you make and accidentally issuing actions, because you looked at something for too long, would result in an unsatisfying experience.</span></span>
<span data-ttu-id="e8a95-144">因此，我們建議您合併眼睛與 *語音命令* 、 *手勢手勢* 、 *按鈕點擊* 或延伸停留，以觸發選取目標 (如需詳細資訊，請參閱 [眼睛和認可](gaze-and-commit-eyes.md)) 。</span><span class="sxs-lookup"><span data-stu-id="e8a95-144">Therefore, we recommend combining eye-gaze with a *voice command* , *hand gesture* , *button click* or extended dwell to trigger the selection of a target (for more information see [eye-gaze and commit](gaze-and-commit-eyes.md)).</span></span>
<span data-ttu-id="e8a95-145">此解決方案也可讓使用者自由地查看，而不會因為 involuntarily 觸發某個東西而不會產生任何負荷。</span><span class="sxs-lookup"><span data-stu-id="e8a95-145">This solution also allows for a mode in which the user can freely look around without being overwhelmed by involuntarily triggering something.</span></span> <span data-ttu-id="e8a95-146">當您在查看目標時設計視覺和聽覺的意見反應時，也應該考慮這個問題。</span><span class="sxs-lookup"><span data-stu-id="e8a95-146">This issue should also be considered when designing visual and auditory feedback when looking at a target.</span></span>
<span data-ttu-id="e8a95-147">不要讓使用者使用立即的彈出效果或暫留音效。</span><span class="sxs-lookup"><span data-stu-id="e8a95-147">Try not to overwhelm the user with immediate pop-out effects or hover sounds.</span></span> <span data-ttu-id="e8a95-148">奧妙是索引鍵。</span><span class="sxs-lookup"><span data-stu-id="e8a95-148">Subtlety is key.</span></span> <span data-ttu-id="e8a95-149">我們將在討論 [設計建議](eye-gaze-interaction.md#design-recommendations)時，討論下面的一些最佳作法。</span><span class="sxs-lookup"><span data-stu-id="e8a95-149">We will discuss some best practices for this further below when talking about [design recommendations](eye-gaze-interaction.md#design-recommendations).</span></span>

- <span data-ttu-id="e8a95-155">**按一下前保留：** 針對快速目標選取，研究已顯示使用者的眼睛可在結束手動按一下 (例如，) 的情況下繼續進行。</span><span class="sxs-lookup"><span data-stu-id="e8a95-155">**Leave before click:** For quick target selections, research has shown that a user's eye-gaze can move on before concluding a manual click (e.g., an air tap).</span></span> <span data-ttu-id="e8a95-156">因此，必須支付一些特殊的注意，才能以較慢的控制項 (輸入（例如語音、手、控制器) ）來同步處理快速眼睛信號。</span><span class="sxs-lookup"><span data-stu-id="e8a95-156">Hence, special attention must be paid to synchronizing the fast eye-gaze signal with slower control input (e.g., voice, hands, controller).</span></span>

<span data-ttu-id="e8a95-160">您的設計若要為使用者營造出愉快而舒適的體驗，我們建議目標的視角至少應為 2°，最好是更大。</span><span class="sxs-lookup"><span data-stu-id="e8a95-160">For your design, to create a pleasant and comfortable experience for your users, we recommend that targets should be at least 2° in visual angle, preferably larger.</span></span>

## <a name="design-recommendations"></a><span data-ttu-id="e8a95-166">設計建議</span><span class="sxs-lookup"><span data-stu-id="e8a95-166">Design recommendations</span></span>
<span data-ttu-id="e8a95-167">以下是一份特定設計建議的清單，這些建議是以眼睛眼輸入所述的優點和挑戰為基礎：</span><span class="sxs-lookup"><span data-stu-id="e8a95-167">The following is a list of specific design recommendations based on the described advantages and challenges for eye-gaze input:</span></span>

1. <span data-ttu-id="e8a95-168">**眼睛與列印頭不同：：**</span><span class="sxs-lookup"><span data-stu-id="e8a95-168">**Eye-gaze is not the same as Head-gaze:**</span></span>
    - <span data-ttu-id="e8a95-169">**考慮快速但不完全的眼睛移動是否符合您的輸入工作：** 雖然快速且不齊的眼睛都非常適合在我們的觀點來快速選取目標，但是較不適合需要平滑輸入軌跡的工作 (例如，繪製或 encircling 批註) 。</span><span class="sxs-lookup"><span data-stu-id="e8a95-169">**Consider whether fast yet ragged eye movements fit your input task:** While our fast and ragged eye movements are great at quickly selecting targets across our field of view, it is less applicable for tasks that require smooth input trajectories (e.g., drawing or encircling annotations).</span></span> <span data-ttu-id="e8a95-170">在此情況下，手部或頭部指向應該較為合用。</span><span class="sxs-lookup"><span data-stu-id="e8a95-170">In this case, hand or head pointing should be preferred.</span></span>
  
    - <span data-ttu-id="e8a95-171">**避免將某個東西直接附加至使用者的眼睛 (例如，滑杆或游標) 。**</span><span class="sxs-lookup"><span data-stu-id="e8a95-171">**Avoid attaching something directly to the user’s eye-gaze (e.g., a slider or cursor).**</span></span>
<span data-ttu-id="e8a95-172">如果是資料指標，這可能會導致「fleeing 資料指標」效果，因為投射的眼睛信號中有稍微位移。</span><span class="sxs-lookup"><span data-stu-id="e8a95-172">In case of a cursor, this may result in a "fleeing cursor" effect due to slight offsets in the projected eye-gaze signal.</span></span> <span data-ttu-id="e8a95-173">在滑杆的情況下，它可能會與使用眼睛控制滑杆的雙角色發生衝突，同時也想要檢查物件是否位於正確位置。</span><span class="sxs-lookup"><span data-stu-id="e8a95-173">In case of a slider, it can conflict with the double role of controlling the slider with your eyes while also wanting to check whether the object is at the correct location.</span></span> <span data-ttu-id="e8a95-174">在滑杆的範例中，使用眼睛搭配手勢會更有意義。</span><span class="sxs-lookup"><span data-stu-id="e8a95-174">For the example of the slider, it makes more sense to use eye-gaze in combination with hand gestures.</span></span> <span data-ttu-id="e8a95-175">這表示使用者可以快速且輕鬆地在許多滑杆之間切換，進而提高其手並捏合其拇指和食指，以抓取並移動它。</span><span class="sxs-lookup"><span data-stu-id="e8a95-175">This means that the user could quickly and effortlessly switch among a number of sliders, raising up their hand and pinching their thumb and index finger to grab and move it.</span></span> <span data-ttu-id="e8a95-176">當您釋放縮小時，滑杆會停止移動。</span><span class="sxs-lookup"><span data-stu-id="e8a95-176">When the pinch is released, the slider stops moving.</span></span> <span data-ttu-id="e8a95-177">簡單地說，使用者可能會被淹沒和破壞，特別是如果該使用者的信號不精確。</span><span class="sxs-lookup"><span data-stu-id="e8a95-177">In a nutshell, users could become overwhelmed and distracted, especially if the signal is imprecise for that user.</span></span> 
  
2. <span data-ttu-id="e8a95-178">將 **眼睛與其他輸入結合：** 眼睛追蹤與其他輸入（例如手勢、語音命令或按鈕按下）的整合提供數個優點：</span><span class="sxs-lookup"><span data-stu-id="e8a95-178">**Combine eye-gaze with other inputs:** The integration of eye tracking with other inputs, such as hand gestures, voice commands or button presses, provides several advantages:</span></span>
    - <span data-ttu-id="e8a95-179">**允許免費觀察：** 假設我們眼的主要角色是觀察我們的環境，請務必讓使用者不需要觸發任何 (視覺效果、聽覺等 ) 意見反應或動作，就能進行查詢。</span><span class="sxs-lookup"><span data-stu-id="e8a95-179">**Allow for free observation:** Given that the main role of our eyes is to observe our environment, it is important that users are allowed to look around without triggering any (visual, auditory, etc.) feedback or actions.</span></span> 
    <span data-ttu-id="e8a95-180">將眼睛追蹤與另一個輸入控制項結合，可讓您在眼睛追蹤觀察和輸入控制模式之間進行順暢的轉換。</span><span class="sxs-lookup"><span data-stu-id="e8a95-180">Combining eye tracking with another input control allows smooth transitioning between eye tracking observation and input control modes.</span></span>
  
    - <span data-ttu-id="e8a95-181">**強大的內容提供者：** 在說出語音命令或執行手勢時，使用使用者所查看之位置和內容的相關資訊，可讓您順暢地將整個現場的輸入。</span><span class="sxs-lookup"><span data-stu-id="e8a95-181">**Powerful context provider:** Using information about where and what the user is looking at while saying a voice command or performing a hand gesture allows seamlessly channeling the input across the field-of-view.</span></span> <span data-ttu-id="e8a95-182">例如：說「 _放入該處_ 」以快速且流暢在場景中選取並放置全像投影，只要查看目標及其預定目的地即可。</span><span class="sxs-lookup"><span data-stu-id="e8a95-182">For example: Say _"put that there"_ to quickly and fluently select and position a hologram across the scene by simply looking at a target and its intended destination.</span></span> 

    - <span data-ttu-id="e8a95-183">**需要同步處理的輸入：** 將快速眼睛的移動與更複雜的額外輸入（例如長聲音命令或手手勢）結合，會在其他輸入命令完成和辨識之前，有使用者已繼續尋找的風險。</span><span class="sxs-lookup"><span data-stu-id="e8a95-183">**Need for synchronizing multimodal inputs:** Combining rapid eye movements with more complex additional inputs, such as long voice commands or hand gestures, bears the risk that the user already continues to look around before the additional input command is finished and recognized.</span></span> <span data-ttu-id="e8a95-184">因此，如果您建立自己的輸入控制項 (例如) 自訂手勢，請務必記錄此輸入的驗證或大約的持續時間，以將它與使用者過去查看的內容相互關聯。</span><span class="sxs-lookup"><span data-stu-id="e8a95-184">Hence, if you create your own input controls (e.g., custom hand gestures), make sure to log the onset of this input or approximate duration to correlate it with what a user had looked at in the past.</span></span>
    
3. <span data-ttu-id="e8a95-185">**目視追蹤輸入的微妙意見反應：** 在查看目標以表示系統如預期運作，但應該保持微妙的情況時，提供意見反應會很有用。</span><span class="sxs-lookup"><span data-stu-id="e8a95-185">**Subtle feedback for eye tracking input:** It's useful to provide feedback when a target is looked at to indicate that the system is working as intended but should be kept subtle.</span></span> <span data-ttu-id="e8a95-186">這包括緩慢的混色、進出、視覺反白顯示或執行其他微妙的目標行為，例如緩慢的動作（例如稍微增加目標大小），表示系統已正確偵測到使用者正在查看目標，而不需要中斷使用者目前的工作流程。</span><span class="sxs-lookup"><span data-stu-id="e8a95-186">This can include slowly blending, in and out, visual highlights or perform other subtle target behaviors, such as slow motions, such as slightly increasing the target size, to indicate that the system correctly detected that the user is looking at a target without unnecessarily interrupting the user’s current workflow.</span></span> 

4. <span data-ttu-id="e8a95-187">**避免將非自然眼移動強制為輸入：** 請勿強制使用者執行特定的眼睛移動 (注視手勢) 在您的應用程式中觸發動作。</span><span class="sxs-lookup"><span data-stu-id="e8a95-187">**Avoid enforcing unnatural eye movements as input:** Do not force users to perform specific eye movements (gaze gestures) to trigger actions in your application.</span></span>

5. <span data-ttu-id="e8a95-188">**Imprecisions 的帳戶：** 我們區分兩種類型的 imprecisions，這對使用者來說很明顯：位移和抖動。</span><span class="sxs-lookup"><span data-stu-id="e8a95-188">**Account for imprecisions:** We distinguish two types of imprecisions which are noticeable to users: offset and jitter.</span></span> <span data-ttu-id="e8a95-189">解決位移最簡單的方式，就是提供夠大的目標來與互動。</span><span class="sxs-lookup"><span data-stu-id="e8a95-189">The easiest way to address an offset is to provide sufficiently large targets to interact with.</span></span> <span data-ttu-id="e8a95-190">建議您使用大於2°的視覺角度作為參考。</span><span class="sxs-lookup"><span data-stu-id="e8a95-190">It is suggested that you use a visual angle greater than 2° as a reference.</span></span> <span data-ttu-id="e8a95-191">比方說，當您延展 arm 時，縮圖大約是以視覺角度表示的2°。</span><span class="sxs-lookup"><span data-stu-id="e8a95-191">For instance, your thumbnail is about 2° in visual angle when you stretch out your arm.</span></span> <span data-ttu-id="e8a95-192">據此我們提供下列指導方針：</span><span class="sxs-lookup"><span data-stu-id="e8a95-192">This leads to the following guidance:</span></span>
    - <span data-ttu-id="e8a95-193">不要強制使用者選取 [小型目標]。</span><span class="sxs-lookup"><span data-stu-id="e8a95-193">Do not force users to select tiny targets.</span></span> <span data-ttu-id="e8a95-194">研究顯示，如果目標夠大，而且系統設計得當，使用者就可以輕鬆且神奇地描述其互動。</span><span class="sxs-lookup"><span data-stu-id="e8a95-194">Research has shown that if targets are sufficiently large, and that the system is designed well, users describe their interactions as effortless and magical.</span></span> <span data-ttu-id="e8a95-195">如果目標太小，則使用者會反映體驗令人疲累又洩氣。</span><span class="sxs-lookup"><span data-stu-id="e8a95-195">If targets become too small, users describe the experience as fatiguing and frustrating.</span></span>
